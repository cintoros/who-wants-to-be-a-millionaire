Project work at the University of Applied Sciences FHNW

This is a project work within the module: Web Engineering. I was allowed to submit an own project idea, which I should implement with Grails 4.

As you can see from the title, the general idea was based on a "Who Wants to be a Millionaire" game.

More about the project can be found in the following chapters.

General conditions

The choice of front-end technology was up to us and is ideally cast in "Grails profiles". I have chosen the standard web profile, which works with Groovy Server Pages, short gsp.

Other profiles like React, Angular and many more would have been possible.

Time

It was a lot of fun to work on this project! Even though I spent far more than 40 hours invested in it, I could try out a great framework and at the same time Get to know the programming language Groovy. This is valuable experience, which I will use well in the future can be.

Instruction

The most important application links are listed below.

  • /play - Play the Game. Der Klassiker. Spielen sie durch 15 knifflige Fragen allerhand Themengebieten durch und werden Sie zum Quizkönig.
  • /moderator - Be the Moderator, show/hide correct answers and clear questions. Do you want to have a cozy game night with the family? Then connect your end device to a TV and feel like Günther Jauch. Discuss the questions, display the right and wrong answers. Fun is guaranteed and everything can be played at your pace.
  • /login - Login / Logout as a player / admin
  • /question - Create, Retrieve, Update and Delete Questions as an admin

Implemented Features

  • Game: Play the Game and get through 15 random questions
  • Game: Load a default set of questions from JSON
  • Game: Be the moderator! This functionality allows every player/admin to be the quiz moderator, in order to play with the family on one screen (e.g. connected to the house tv).
  • Questions: create, update and delete questions (admin only)
  • Login / Logout as a player (default username: player password: player123)
  • Login / Logout as an admin (default username: admin password: admin123)
  • Security: Password rule constraints
  • Security: Passwords are hashed in the db.
  • Security Role: Admin can create, update and delete questions
  • Security Role: Player cannot create, update and delete questions
  • Security Role: Admin can play the game normally
  • Security Role: Player can play the game normally
  • Security Role: Admin can be the moderator
  • Security Role: Player can be the moderator

This is what I'm proud of

I would like to point out a few points that I have implemented well in terms of implementation.

Anti Cheat Detection

If a player looses the game, presses the back button to "reanswer" his question, the anti cheat detection get's triggered and the game is aborted.

Zoom Logo

Who wants to be a millionaire logo on the home page has an "on hover" zoom animation. Actually this has taken a lot of time, but it was worth it, because the page looks less static now.

Design and layout

Design and layout in general is very well done. Bootstrap 4 runs through the application and gives it a uniform and stable touch.

50/50 wild cards

The 50/50 Joker is available in game mode and can only be used once per game. Thereby two wrong answers are eliminated from the current questions.

Responsive website

The website in general is dynamic: the 50/50 joker disappears as soon as it is played. Dynamic button names are also available in the moderator view. The button changes from "Show Answer" to "Hide Answer" as soon as it is clicked. "Clear Answer" sets the button always go back to "Show Answer" as well. I have experimented a lot with Javascript for these features and came up with very good results.

Initial Question Catalogue

Via an online JSON-API questions can be loaded dynamically (with fail-over on local questions) and users can directly open the app and play!

Valid HTML and CSS

Every HTML and CSS page has been validated with "https://validator.w3.org/nu/" and contains no errors anymore. The moderator view was the most difficult view to provide without errors.

Projektarbeit bei der FHNW

Dies ist eine Projektarbeit im Rahmen des Moduls: Web Engineering. Ich durfte eine eigene Projektidee einreichen, die ich mit Grails 4 umsetzen sollte.

Wie dem Titel zu entnehmen ist, habe ich mich, von der Allgemeinen Idee her, an ein Wer Wird Millionär Spiel angelehnt.

Mehr zum Projekt ist in den fortfolgenden Kapiteln zu entnehmen.

Rahmenbedingungen

Die Wahl der Frontend Technologie war uns freigestellt und ist ideal in "Grails Profilen" gegossen. Ich habe mich fürs Standard Web-Profil entschieden, welches mit Groovy Server Pages, kurz gsp arbeitet.

Andere Profile wie React, Angular und viele weitere wären denkbar gewesen.

Anleitung

Untenstehend sind die wichtigsten Applikationslinks aufgeführt.

  • /play - Play the Game. Der Klassiker. Spielen sie durch 15 knifflige Fragen allerhand Themengebieten durch und werden Sie zum Quizkönig.
  • /moderator - Be the Moderator, show/hide correct answers and clear questions. Wollen Sie sich einen gemütlichen Spieleabend in der Familie machen? Schliessen Sie dafür ihr Endgerät beispielsweise an einen Fernseher an und fühlen Sie sich wie Günther Jauch. Diskutieren Sie die Fragen aus, zeigen Sie die richtigen und falschen Antworten an. Alles in Ihrem Tempo.
  • /login - Login/Logout as a player / admin
  • /question - Create, Retrieve, Update and Delete Questions as an admin

Implemented Features

  • Game: Play the Game and get through 15 random questions
  • Game: Load a default set of questions from JSON
  • Game: Be the moderator! This functionality allows every player/admin to be the quiz moderator, in order to play with the family on one screen (e.g. connected to the house tv).
  • Questions: create, update and delete questions (admin only)
  • Login / Logout as a player (default username: player password: player123)
  • Login / Logout as an admin (default username: admin password: admin123)
  • Security: Password rule constraints
  • Security: Passwords are hashed in the db.
  • Security Role: Admin can create, update and delete questions
  • Security Role: Player cannot create, update and delete questions
  • Security Role: Admin can play the game normally
  • Security Role: Player can play the game normally
  • Security Role: Admin can be the moderator
  • Security Role: Player can be the moderator

Darauf bin ich stolz

Ich möchte auf ein paar Punkte hinweisen, auf die ich Realisierungstechnisch gut umgesetzt habe.

  • Wer Wird Millionär Logo auf der Startseite mit Zoom-Animation. Tatsächlich hat das viel Zeit in Anspruch genommen. Das hat sich aber gelohnt, weil die Seite damit weniger statisch wirkt.
  • Design und Layout im Allgemeinen ist sehr gelungen. Bootstrap 4 zieht sich durch die Applikation und verleiht ihr einen einheitlichen und stabilen Touch.
  • 50/50 Joker ist im Spiel-Modus verfügbar und kann pro Spiel nur einmal eingesetzt werden.
  • Dynamische Buttonnamen in Moderator Ansicht: Button wechselt von "Show Answer" zu "Hide Answer" bei Click. "Clear Answer" setzt den Button ebenfalls zurück zu "Show Answer". Ich habe bei diesem Feature viel mit Javascript experimentiert und kam auf zufriedenstellende Ergebnisse.
  • Über ein JSON-API sind Fragen dynamisch ladbar und Benutzer können direkt bei Startup losspielen!
